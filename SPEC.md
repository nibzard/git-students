## Part A — Multiple Choice (1 point each)

1. What is **Git** primarily used for?
   A. Hosting websites
   B. Version control (tracking changes in files)
   C. Writing code faster
   D. Running code online

2. What is **GitHub** primarily used for?
   A. A cloud service to store and collaborate on Git repositories
   B. A programming language
   C. A text editor
   D. A database

3. Which command creates a **new local Git repository** in the current folder?
   A. `git start`
   B. `git init`
   C. `git clone`
   D. `git new`

4. Which command shows the current status (staged/unstaged changes)?
   A. `git status`
   B. `git check`
   C. `git log`
   D. `git diff --all`

5. What does `git add .` do?
   A. Uploads files to GitHub
   B. Stages changes in the current directory for commit
   C. Deletes untracked files
   D. Creates a new branch

6. What does `git commit` do?
   A. Saves a snapshot of staged changes in the local repo
   B. Uploads your files to GitHub
   C. Downloads changes from GitHub
   D. Removes changes

7. Which command is typically used to upload local commits to a remote repository?
   A. `git upload`
   B. `git push`
   C. `git send`
   D. `git publish`

8. Which command downloads commits from a remote **and merges them** into your current branch?
   A. `git pull`
   B. `git fetch`
   C. `git download`
   D. `git merge --remote`

9. What is a **branch** in Git?
   A. A copy of your computer
   B. A separate line of development
   C. A folder on GitHub
   D. A backup of all files

10. What is a **merge conflict**?
    A. GitHub server is down
    B. Two branches changed the same part of a file in incompatible ways
    C. Your password is wrong
    D. You forgot to commit

11. In the typical local workflow, what comes **right after** `git add`?
    A. `git pull`
    B. `git commit`
    C. `git clone`
    D. `git fork`

12. What does `origin` usually refer to in Git?
    A. Your current branch name
    B. The default name for the remote you cloned from (often your fork on GitHub)
    C. The first commit in the project
    D. A backup folder Git creates automatically

13. What is the purpose of a **`.gitignore`** file?
    A. It encrypts your repository so others can’t see it
    B. It tells Git which files/folders to ignore so they won’t be tracked/committed
    C. It automatically fixes merge conflicts
    D. It syncs your repo with GitHub every minute

14. Which statement best describes `git clone`?
    A. Copies a remote repository to your computer for the first time
    B. Uploads your local repo to GitHub
    C. Deletes a repository
    D. Renames a repository

15. Which statement best describes a **Pull Request (PR)**?
    A. A way to delete commits from history
    B. A request to merge your changes (usually from a branch/fork) into another repo/branch
    C. A command that downloads updates from GitHub
    D. A tool that automatically writes commit messages

16. You edit `README.md`, and a new file `notes.txt` appears that you **do not want in the repo**. What is the best Git/GitHub concept for this?
    A. Add `notes.txt` to `.gitignore` so it’s not tracked
    B. Put `notes.txt` into the `.git` folder
    C. Commit it and delete it later
    D. Rename it to `README2.md`

17. In a “teacher repo → student fork → PR back” workflow, what is the **most common reason** students’ PRs get messy or conflicts increase?
    A. They forget to star the repository on GitHub
    B. They work on their fork without syncing it with the teacher repo first
    C. They use too many commit messages
    D. They push their project into the `README.md` file
